Principles and best practice

As a Clan we have pulled together a set of principles and best practices which we use for Interaction Design to provide consistency and guidance to how we apply design rationale and practices leading to great interaction design and a great user experience.

Principles list

Design for ALL users; their needs, behaviours and behavioural science

...so that we are meeting actual user needs for all users

Understand all users, their needs, behaviours and pain points

Best practice guidance:

  • Use needs-based personas
  • Consider both user needs (including accessibility needs) and business needs
  • Understand the research insights
  • Factor in user behaviours
  • Fix or mitigate user pain points
  • Think about what's missing (to feed into UR/workshops)
Design accessibly from the start

Best practice guidance:

  • Use sufficient colour contrast
  • Use industry recommended sizes for interactive elements
  • Design structure (headings, lists...)
  • Visually indicate the page focus
  • Avoid images of text
  • Do not use colour alone to convey meaning
Follow Gestalt laws

Best practice guidance:

  • Law of proximity
  • Law of closure
  • Law of similarity
  • Law of continuity
Reduce cognitive load

Best practice guidance:

  • Prefill data
  • Use summary pages to playback to users
  • Make it easy to compare data from different sources
  • Miller's law (chunking) to help memory limitations
  • Allow reference material to open in a new window
Support users / preventative IxD

Best practice guidance:

  • Allow users to undo actions
  • Think error recovery and error prevention
  • Provide confirmation of actions e.g. save and delete
  • Make users explicity confirm destructive actions e.g. are you sure you want to delete?
  • Provide logical options for checkboxes / radio buttons
  • Add helpful context e.g. hint text
  • Include helpful validation messages

Design for simplicity

...so the services are intuiative and easy to use
Understand user behaviours and make users do less

Best practice guidance:

  • Ask yourself / understand 'how' the user does xyz
  • Ask yourself / understand 'why' the user does xyz
  • Pre-populate, auto filter, remove redundancies
  • Avoid unnecessary or extra interactions / provide short cuts
  • Reduce scrolling (use back to top, anchor links)
  • Fitt's law - time relative to size and distance
  • Reduce repetitive interactions, (multiple instead of individual)
Don't make users think

Best practice guidance:

  • Use of summary screens before submitting
  • Recognition rather than recall, have access to / what users need where they need it
  • Add helpful contextual information / guidance
  • Use of all elements on the page headings, hint text labels
  • Map to existing mental models
  • Keep focus (not cluttered, provide affordance, clear call to actions)

Design in context

...so users expectations are set and they know where they are, where they are going, what they need and what they will be doing
Understand context and provide context

Best practice guidance:

  • Avoid dead-ends
  • Explain what will happen next
  • Tell users what they need
  • Indicate timescales
  • Consider end-2-end journey
  • Consider environment, IT confidence, pressures and constraints
  • Provide timely system feedback

Design consistently

...so interactions for the services are predictable and learnable thereby saving users time and effort
Design a seamless experience

Best practice guidance:

  • Re-use patterns / code where possible and appropriate to conform with user expectations
  • Avoid bespoke designs where possible
  • Make sure systems look / behave the same
  • Maintain documentation e.g. Design histories, design decisions, other IxD dcoumentation

Design meaningful interactions

...so that everything is grounded in research / design rationale of proven interactions
Design with data / design logic

Best practice guidance:

  • Designs are testing hypothesis
  • Features are fulfilling a user need
  • Design rationale exists for design decisions
  • Remove unnecessary components which are not intent driven or do not add value
  • Follow User Centred Design best practice
